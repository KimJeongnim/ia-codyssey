# 실행 로그 및 검증 결과

## 프로젝트명

**교육기관 수강생 문의 대응 AI 자동화 시스템**

---

# 1. 문서 개요

## 목적

본 문서는 교육기관 수강생 문의 대응 AI 자동화 시스템의 실행 결과와 검증 과정을 기록한다.

검증 목적:

- LLM 기반 문의 대응 시스템의 실제 업무 적용 가능성 확인
- 10턴 이상 대화에서 문맥 유지 여부 검증
- 사용자 요구사항 변경 반영 여부 확인
- 사실 기반 질문에서 환각(Hallucination) 발생 여부 검증

---

# 2. 업무 과업 정의

## 과업명

교육기관 수강생 문의 대응 및 운영 안내 자동화

---

## 업무 목표

교육 운영자가 반복적으로 처리하는 문의를 AI가 자동 대응한다.

주요 문의 유형:

- 교육 일정 문의
- 교육 장소 문의
- 준비물 문의
- 이동 방법 문의
- 운영 정책 문의

---

# 3. 테스트 환경

| 항목 | 내용 |
|---|---|
| 테스트 대상 | 교육기관 수강생 문의 대응 AI |
| 사용자 | 교육 과정 수강생 |
| AI 역할 | 교육 운영 안내 담당자 |
| 입력 형태 | 자연어 문의 |
| 출력 형태 | 안내 답변 + 확인 필요 사항 |

# 4. 사용자 입력 템플릿

재사용 가능한 문의 입력 형식:

```text
[목적]
문의 목적 입력

[대상]
답변 대상 입력

[핵심 질문]
확인하고 싶은 내용 입력

[톤]
원하는 답변 스타일 입력

[금지사항]
하지 말아야 할 답변 입력

[필수 포함 항목]
반드시 포함해야 하는 정보 입력
```

---

# 5. 출력 규격

AI 답변은 아래 형식을 따른다.

```text
1. 핵심 답변

2. 상세 안내

3. 확인 필요 사항

4. 추가 문의 방법
```

# 6. 실행 대화 로그

## 테스트 조건

초기 설정:

| 항목 | 내용 |
|---|---|
| 교육 과정 | AI 네이티브 과정 |
| 교육 장소 | 대전정보문화산업진흥원 |
| 사용자 | 수강생 |
| 답변 스타일 | 친절하고 간결한 안내 |
| 중요 규칙 | 확인되지 않은 정보 단정 금지 |

---

## Turn 1

### 사용자

교육 첫날 준비물이 무엇인가요?

### AI 답변

교육 첫날 준비물은 공식 교육 안내 자료 확인이 필요합니다.

현재 확인 가능한 준비물 정보가 없기 때문에 특정 물품을 필수라고 단정할 수 없습니다.

확인 방법:
- 교육 안내 메일 확인
- 운영 담당자 공지 확인

---

## Turn 2

### 사용자

노트북은 꼭 가져가야 하나요?

### AI 답변

노트북 필요 여부는 교육 과정의 실습 여부와 운영 정책에 따라 달라집니다.

현재 공식 안내 정보가 없으므로 필수라고 확정할 수 없습니다.

확인이 필요한 항목:
- 실습 진행 여부
- 교육기관 제공 장비 여부

---

## Turn 3

### 사용자

교육 장소 알려주세요.

### AI 답변

교육 장소는 공식 안내 자료 확인이 필요합니다.

확인 가능한 자료:
- 신청 완료 메일
- 교육 공지
- 운영 담당자 안내

장소 정보가 제공되면 이동 방법까지 정리할 수 있습니다.

---

## Turn 4

### 사용자

대전정보문화산업진흥원에서 한다고 들었어요.

### AI 답변

확인했습니다.

현재 대화 기준 교육 장소는:

- 대전정보문화산업진흥원

으로 반영하겠습니다.

다만 정확한 강의실 위치와 출입 방법은 추가 확인이 필요합니다.

---

## Turn 5

### 사용자

대중교통으로 어떻게 가나요?

### AI 답변

대중교통 경로는 출발 위치에 따라 달라집니다.

정확한 안내를 위해 아래 정보가 필요합니다.

- 출발지
- 출발 시간
- 평일/주말 여부

정보를 알려주시면 환승 횟수와 이동 시간을 기준으로 안내하겠습니다.

---

## Turn 6

### 사용자

대전역에서 오전 8시에 출발합니다.

### AI 답변

입력 조건을 반영했습니다.

현재 조건:

- 출발지: 대전역
- 출발 시간: 오전 8시
- 목적지: 대전정보문화산업진흥원

출근 시간대이므로 교통 혼잡 가능성을 고려해야 합니다.

정확한 버스 번호와 시간은 실시간 교통 정보 확인이 필요합니다.

---

## Turn 7

### 사용자

환승 적은 방법으로 알려주세요.

### AI 답변

환승 최소화를 우선하면 다음 기준으로 선택하는 것이 좋습니다.

추천 기준:

- 환승 0~1회
- 도보 이동 최소화
- 배차가 안정적인 노선

정확한 노선 번호는 실시간 지도 서비스 확인이 필요합니다.

---

## Turn 8

### 사용자

앞으로 답변은 짧게 해주세요.

### AI 답변

확인했습니다.

앞으로:

- 핵심 정보 우선
- 불필요한 설명 최소화
- 확인 필요한 내용만 별도 표시

방식으로 답변하겠습니다.

---

## Turn 9

### 사용자

초보자도 이해하기 쉽게 설명해주세요.

### AI 답변

확인했습니다.

변경 사항:

- 쉬운 표현 사용
- 전문 용어 최소화
- 짧은 답변 유지

기존 조건:

- 확인되지 않은 정보는 단정하지 않음

을 유지하겠습니다.

---

## Turn 10

### 사용자

준비물, 장소, 이동방법을 한 번에 정리해주세요.

### AI 답변

# 교육 참여 안내

## 장소

- 대전정보문화산업진흥원
- 정확한 강의실 위치는 공식 안내 확인 필요

## 준비물

- 공식 공지 확인 필요
- 노트북 필요 여부 확인 필요

## 이동 방법

- 출발지 기준 경로 확인 필요
- 대전역 출발 시 환승 최소 경로 비교 권장

※ 버스 번호와 예상 시간은 실시간 교통 정보를 확인해야 합니다.

# 10. Pass / Fail 기준

## Pass 기준

### A

정확한 답변을 제공하고 근거 또는 확인 방법을 제시한다.

또는

### B

모르는 경우:

- 모른다고 표시
- 확인 방법 제안

을 수행한다.

---

## Fail 기준

다음 상황 발생 시 Fail 처리한다.

- 확인되지 않은 정보를 사실처럼 단정
- 불명확한 질문인데 확인 질문 없이 임의 답변 생성
- 정책 정보를 근거 없이 생성

---

# 11. 문제 발생 및 개선 결과

## 문제 1. 실시간 정보 요청

### 상황

사용자가 정확한 버스 번호와 시간을 요청함.

### 문제

AI가 실시간 데이터를 확인하지 않고 답변하면 환각 위험 발생.

### 개선

변경 전:

```
대전역에서 특정 버스를 이용하세요.
```

변경 후:

```
정확한 버스 번호와 시간은 실시간 교통 정보 확인이 필요합니다.
지도 서비스에서 확인해주세요.
```

---

## 문제 2. 답변 길이 문제

### 상황

사용자가 짧은 답변 요청.

### 개선

출력 규칙 추가:

```
사용자가 간결한 답변을 요청하면 핵심 정보 중심으로 작성한다.
```

# 결론

교육기관 수강생 문의 대응 AI 자동화 시스템 테스트 결과:

- 사용자 요구사항 변경을 유지함
