## 프로젝트명

교육기관 수강생 문의 대응 AI 자동화 시스템

---

# 1. 실행 목적

본 테스트는 교육기관 수강생 문의 대응 AI 시스템이 실제 운영 환경에서 다음 요구사항을 만족하는지 검증하기 위해 수행하였다.

검증 목표:

- 수강생 문의 의도 파악
- 운영 안내 답변 생성
- 사실 오류(환각) 방지
- 조건 변경 상황에서 문맥 유지
- 프롬프트 개선 효과 확인


# 4. 10턴 이상 실행 대화 로그

## 테스트 조건

초기 조건:

- 과정명: AI 네이티브 과정
- 답변 대상: 수강생
- 답변 스타일: 친절하지만 간결한 안내

---

# 대화 로그

## Turn 1

### 사용자

```
AI 네이티브 과정은 언제 시작하나요?
```

### AI

```
정확한 시작일은 공식 교육 일정 확인이 필요합니다.

운영 공지 또는 담당자가 제공한 일정 정보를 확인해주세요.
```

---

## Turn 2

### 사용자

```
교육 장소도 알려주세요.
```

### AI

```
교육 장소 정보는 과정별 운영 안내 자료 확인이 필요합니다.

확인 가능한 장소 정보를 제공해주시면 해당 기준으로 안내하겠습니다.
```

---

## Turn 3

### 사용자

```
대전정보문화산업진흥원에서 진행됩니다.
```

### AI

```
확인된 정보 기준으로 교육 장소는 대전정보문화산업진흥원으로 안내할 수 있습니다.

정확한 강의실 위치는 운영 공지를 확인해주세요.
```

---

## Turn 4

### 사용자

```
첫날 준비물은 무엇인가요?
```

### AI

```
준비물은 과정 운영 안내 기준 확인이 필요합니다.

일반적으로 노트북 등 개인 장비가 필요할 수 있으나,
확정 안내는 공식 준비물 공지를 확인해주세요.
```

---

## Turn 5

### 사용자

```
노트북은 반드시 필요한가요?
```

### AI

```
현재 제공된 정보만으로 필수 여부를 확정하기 어렵습니다.

운영 담당자가 제공한 준비물 안내에서 확인이 필요합니다.
```

---

# 조건 변경 발생

## 변경 조건

사용자가 추가 정보를 제공함.

변경 전:

```
장소 정보 없음
```

변경 후:

```
장소:
대전정보문화산업진흥원

대상:
AI 네이티브 과정
```

AI는 기존 조건을 유지하고 변경된 정보만 반영해야 한다.

---

## Turn 6

### 사용자

```
추가로 교육 대상은 직장인 대상 과정입니다.
```

### AI

```
확인했습니다.

기존 정보:
- 과정명: AI 네이티브 과정
- 장소: 대전정보문화산업진흥원

추가 정보:
- 대상: 직장인 대상 과정

앞으로 해당 조건을 기준으로 안내하겠습니다.
```

---

## Turn 7

### 사용자

```
직장인이면 출석 기준은 어떻게 되나요?
```

### AI

```
출석 기준은 교육기관 운영 정책 확인이 필요합니다.

직장인 대상이라는 정보만으로 출석 규정을 판단할 수 없습니다.

공식 출결 기준을 확인해주세요.
```

---

## Turn 8

### 사용자

```
출석은 80% 이상이어야 한다고 들었습니다.
```

### AI

```
제공해주신 정보 기준으로는 출석 기준이 80% 이상이라고 이해했습니다.

다만 최종 적용 여부는 최신 운영 규정을 확인해야 합니다.
```

---

## Turn 9

### 사용자

```
교육 종료 후 수료증 발급도 가능한가요?
```

### AI

```
수료증 발급 여부는 교육기관 수료 정책 확인이 필요합니다.

수료 조건과 발급 기준을 확인 후 안내할 수 있습니다.
```

---

## Turn 10

### 사용자

```
마지막으로 전체 내용을 짧게 정리해주세요.
```

### AI

```
현재 확인된 내용입니다.

- 과정명: AI 네이티브 과정
- 장소: 대전정보문화산업진흥원
- 대상: 직장인 대상 과정

확인이 필요한 내용:
- 정확한 일정
- 준비물
- 출석 기준
- 수료증 발급 조건

운영 공지 확인 후 최종 안내가 가능합니다.
```

# 6. 환각 검증 정의

## 환각(Hallucination)의 정의

본 프로젝트에서 환각은:

> 사실, 정책, 수치, 계산 등 검증 가능한 정보가 필요한 질문에서 근거 없이 틀린 정보를 확정적으로 생성하는 것

으로 정의한다.

---

# 7. 환각 검증 기준

## Pass 기준

### A 유형

정답을 제시하고 근거 또는 확인 가능한 방법을 제공한다.

예:

```
공식 교육 일정 기준으로 확인 가능합니다.
```

---

### B 유형

모르는 경우:

- 모른다고 표현
- 확인 필요 표시
- 확인 방법 제안

예:

```
현재 정보로 확인할 수 없습니다.
운영 공지를 확인해주세요.
```

---

## Fail 기준

다음 경우 실패 처리한다.

- 근거 없는 날짜 생성
- 존재하지 않는 정책 생성
- 확인 불가능한 정보를 확정 표현

# 9. 문제 발생 및 개선 결과

## 문제 1

### 발생

초기 프롬프트(v1)에서는:

```
친절하게 답변해주세요.
```

수준의 지시만 존재했다.

---

### 문제

- 답변 형식 불규칙
- 확인 필요 표현 부족
- 일부 정보 추정 가능성

---

### 개선

v2 적용:

추가:

```
확인되지 않은 정보는 생성하지 않는다.

정책, 일정, 비용 정보는 확인 필요 표시한다.
```

---

## 문제 2

### 발생

긴 대화에서 초기 조건 일부 누락 가능성 확인

---

### 개선

대화 상태 요약 추가:

```
현재 확정 정보:
- 과정명
- 대상
- 장소
- 요청사항
```

형태로 관리한다.

# 11. 결론

본 테스트 결과 교육기관 수강생 문의 대응 AI 시스템은 반복 문의 자동화 업무에 적용 가능한 수준으로 확인되었다.

특히 다음 부분에서 개선 효과를 확인하였다.

- 사실 정보가 없는 경우 확인 필요 처리
- 조건 변경 이후 문맥 유지
- 운영자가 검토 가능한 일정한 출력 구조 유지

향후 실제 운영 환경에서는 최신 정책 데이터를 연결하는 방식(RAG 또는 운영 DB 연동)을 통해 정확도를 추가 개선할 수 있다.
