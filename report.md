# 🧠 LLM 모델 비교 선정 보고서 (Evaluation-Ready Version)

---

## 1. 업무 과업 정의 (Criterion 1)

### 🎯 Task

대전정보문화산업진흥원까지의 대중교통 이동 전략 생성

---

### 👤 Target User

- AI 교육 수강생
- 대중교통 기반 출퇴근 사용자

---

### 📥 Input

- 출발지
- 시간대
- 환승 제약 조건

---

### 📤 Output

- 경로 2~3개
- 환승 전략
- 시간대별 리스크
- 비교 기반 의사결정 구조

---

## 2. 환각(Hallucination) 정의

본 문서에서 환각은 다음으로 정의한다:

> 실제 존재하지 않는 교통 정보, 시간, 노선, 운영 정책을  
> 사실처럼 단정적으로 생성하는 현상

### 예:
- 존재하지 않는 직행 버스
- 고정되지 않은 막차 시간 단정

---

## 3. 평가 기준 (Criterion 5, 9)

| 기준 | 설명 |
|------|------|
| 정확성 | 실제 교통 구조 일치 |
| 실용성 | 사용 가능 여부 |
| 구조화 | 정보 계층 구조 |
| 간결성 | 중복 제거 |
| 환각 억제 | 사실 오류 최소화 |

---

## 4. 검증 방법 (Criterion 11)

각 모델 출력은 다음 방식으로 검증:

- 실제 교통 구조 논리 비교
- 환승 가능성 검증 (일반 대중교통 기준)
- 시간 정보는 “단정 여부” 기준으로 평가
- 경로 구조 일관성 체크

---

## 9. 결론

LLM 성능 차이는 모델 자체보다:

> 프롬프트 구조 + 평가 기준 설계 + 검증 방식에 의해 결정된다.
