# LLM 모델 비교·선정 보고서

## 프로젝트명

교육기관 수강생 문의 대응 AI 자동화 시스템

---

# 1. 비교 목적

본 문서는 교육기관 운영자가 반복적으로 처리하는 수강생 문의 대응 업무를 자동화하기 위해 적합한 LLM 모델을 비교·평가하고 최종 적용 모델을 선정하는 것을 목적으로 한다.

비교 대상 업무는 다음과 같다.

> 수강생의 교육 일정, 장소, 준비물, 운영 정책 문의에 대해 정확하고 일관된 안내 답변을 생성하는 AI 고객 응대 업무

모델 비교 시 단순 생성 품질이 아닌 실제 업무 적용 관점에서 평가한다.

평가 기준:

- 문의 의도 파악 능력
- 사실 기반 답변 정확성
- 업무용 답변 품질
- 환각 방지 및 확인 요청 능력
- 비용 및 접근성

# 3. 동일 테스트 업무 정의

## 업무 과업

교육기관 수강생 문의 대응 자동화

## 입력 데이터

수강생이 운영 담당자에게 보내는 문의 메시지

예:

```
안녕하세요.
AI 네이티브 과정 관련해서 문의드립니다.

1. 교육 시작일이 언제인가요?
2. 준비물이 필요한가요?
3. 강의장은 어디인가요?

확인 부탁드립니다.
```

---

## 출력 요구사항

AI는 다음 형식으로 답변해야 한다.

```
[답변]

1. 문의 내용 요약

2. 확인된 안내 정보

3. 추가 확인이 필요한 내용

4. 문의자가 해야 할 다음 행동
```

# 4. 동일 입력 프롬프트

모든 모델에 동일한 프롬프트를 입력하였다.

```
당신은 교육기관 운영 지원 AI입니다.

수강생 문의에 답변하세요.

조건:
- 확인되지 않은 정보는 사실처럼 작성하지 않는다.
- 모르는 내용은 확인 필요라고 표시한다.
- 답변은 친절하지만 간결하게 작성한다.
- 운영자가 검토하기 쉬운 형식으로 작성한다.

문의:
"AI 네이티브 과정 교육 장소와 준비물이 궁금합니다.
그리고 첫날 몇 시까지 가야 하는지도 알려주세요."
```

# 8. 최종 선정 모델

## 선정 모델

# GPT-5.4 mini(0.5)

---

# 선정 이유

첫째, 교육기관 문의 대응 업무는 단순 문장 생성보다 **사실 오류 방지와 정책 정보 처리 능력**이 중요하다. GPT-5.4 mini(0.5)는 확인되지 않은 정보를 단정하지 않고 확인 필요 상태를 표현하는 업무 안전성이 높다고 판단하였다.

둘째, 수강생 문의는 일정, 장소, 출결, 환불 정책 등 다양한 유형으로 확장될 수 있다. GPT-5.4 mini(0.5)는 복합 질문을 분류하고 정해진 출력 형식으로 구조화하는 능력이 우수하여 운영 자동화에 적합하다.

셋째, 비용과 성능의 균형 측면에서 대규모 운영 환경에서도 활용 가능한 수준이며, 프롬프트 설계를 통해 무료 모델 대비 안정적인 품질 확보가 가능하다고 판단하였다.

# 9. 비용 제약 상황 대응 방안

만약 무료 모델만 사용해야 하는 경우 다음과 같은 방식으로 품질 저하를 보완한다.

## 프롬프트 개선

- 역할(Persona)을 더 구체적으로 지정
- 답변 형식을 고정
- 금지 표현 정의
- 확인 필요 조건 명시

예:

```
확인되지 않은 일정, 장소, 정책 정보는 절대 생성하지 않는다.
정보가 없으면 반드시 "운영 확인 필요"라고 표시한다.
```

---

## 프로세스 개선

무료 모델 사용 시:

1. AI 1차 답변 생성
2. 규칙 기반 검증
3. 운영자 승인
4. 최종 답변 전달

방식으로 운영한다.

# 10. 최종 결론

교육기관 수강생 문의 대응 자동화 업무에서는 단순 답변 생성보다 정확성, 정책 준수, 환각 방지가 중요하다.

비교 결과 GPT-5.4 mini(0.5)가 가장 높은 종합 점수를 기록했으며, 특히 사실 오류 방지와 업무용 답변 구조화 영역에서 우수한 성능을 보였다.

따라서 본 프로젝트의 기본 운영 모델은GPT-5.4 mini(0.5)를 선정한다.

단, 비용 제약 환경에서는 무료 모델과 강화된 프롬프트 검증 프로세스를 결합하는 방식으로 운영 가능하다.
