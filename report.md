# 1. LLM 모델 비교·선정 보고서

## 1.1 비교 목적

교육기관 수강생 문의 대응 AI 자동화 시스템에 적합한 LLM 모델을 선정하기 위해 동일한 업무 과업 프롬프트를 여러 모델에 적용하고 결과를 비교하였다.

비교 목적:

- 수강생 문의에 대한 정확한 답변 생성 능력 평가
- 교육 운영 정책과 같은 사실 기반 정보 처리 능력 확인
- 모호한 질문에 대한 확인 질문 능력 평가
- 업무 자동화 환경에서 활용 가능한 모델 선정

## 1.3 테스트 업무 과업

### 업무 시나리오

교육기관 운영 담당자를 대신하여 수강생 문의에 답변하는 AI 상담 시스템

### 입력 조건

AI는 다음 역할을 수행한다.

- 교육 일정 안내
- 장소 안내
- 준비물 안내
- 출결 정책 안내
- 환불 및 운영 정책 문의 대응
- 모르는 정보는 추측하지 않고 확인 요청

# 1.4 동일 테스트 프롬프트

모든 모델에 동일한 입력을 적용하였다.

```
당신은 교육기관 수강생 문의 대응 AI 운영 매니저입니다.

목표:
수강생 문의에 정확하고 친절하게 답변합니다.

규칙:
1. 확인되지 않은 사실은 단정하지 않습니다.
2. 정책, 일정, 비용 등 변경 가능한 정보는 확인 필요 여부를 표시합니다.
3. 질문이 모호하면 필요한 정보를 먼저 요청합니다.
4. 답변은 핵심 내용과 근거 중심으로 제공합니다.

문의:
"AI 네이티브 과정 환불 가능한가요?"
```

# 1.8 최종 선정 모델

## 선정 모델

# GPT-5.4 mini(x0.5)

---

## 선정 이유

첫째, 교육기관 문의 대응 업무는 단순한 대화 생성보다 **정확한 정책 처리와 오류 방지**가 중요하다. GPT-5.4 mini(x0.5)는 확인되지 않은 정보를 단정하지 않고 추가 정보를 요청하는 패턴을 안정적으로 수행하였다.

둘째, 실제 운영 환경에서는 다양한 형태의 문의가 발생한다. 같은 질문이라도 사용자가 부족한 정보를 제공할 수 있기 때문에 모호한 입력을 분석하고 필요한 질문을 생성하는 능력이 중요하다.

셋째, 본 프로젝트의 핵심 목표는 친절한 답변보다 **신뢰 가능한 업무 자동화**이다. 따라서 출력 형식 준수, 안전장치 적용, 정책 기반 응답 능력을 종합하여 GPT-5.4 mini(x0.5)를 최종 모델로 선정하였다.

# 1.9 비용 제약 상황 대응 전략

## 무료 모델만 사용해야 하는 경우

무료 모델 사용 시 다음 방식으로 품질 저하를 보완한다.

- 시스템 프롬프트를 더욱 구체적으로 작성
- 자주 발생하는 문의는 Few-shot 예시 추가
- 정책 정보는 별도 문서 검색(RAG) 방식 적용
- 중요한 답변은 운영자 승인 절차 추가

### 무료라서 제한된 점

무료 버전은 사용량 제한, 최신 모델 접근 제한, 긴 문맥 처리 한계 등으로 업무 자동화 품질 관리가 필요하다.

---

# 1.10 모델 선정 결론

최종적으로 교육기관 수강생 문의 대응 AI 자동화 시스템에는 **GPT-5.4 mini(x0.5) 모델을 선정하였다.**

선정 기준은 단순 답변 품질이 아니라 실제 운영 환경에서 필요한 정확성, 정책 처리 안정성, 모호한 질문 대응 능력이다.

특히 교육기관 문의는 일정, 비용, 환불, 출결 등 잘못 안내하면 운영 문제가 발생할 수 있는 정보가 포함된다. 따라서 확인되지 않은 정보를 생성하지 않고 검증 절차를 안내하는 모델이 적합하다.
