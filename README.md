# 업무 자동화 프롬프트 패키지

# 교육기관 수강생 문의 대응 AI 자동화 시스템

---

## 프로젝트 소개

본 프로젝트는 교육기관 운영자가 반복적으로 처리하는 수강생 문의 대응 업무를 LLM(Large Language Model) 기반 AI 시스템으로 자동화하기 위한 업무 자동화 프롬프트 패키지입니다.

교육 일정, 장소, 준비물, 출결, 수료 기준, 운영 정책 등 반복적으로 발생하는 문의를 AI가 처리하고, 사실 확인이 필요한 정보는 운영 담당자가 검토할 수 있도록 설계했습니다.

본 프로젝트는 다음 과정을 포함합니다.

- LLM 모델 비교 및 선정
- 업무 자동화 시스템 설계
- 프롬프트 엔지니어링
- Few-shot 기반 응답 설계
- 환각(Hallucination) 방지 전략
- 실제 대화 실행 및 검증

---

# 1. 업무 과업 정의

## 과업명

**교육기관 수강생 문의 대응 및 운영 안내 자동화**

---

## 업무 목적

교육기관 운영자가 반복적으로 처리하는 수강생 문의를 AI가 자동 응답하도록 구성하여 다음 목표를 달성합니다.

- 반복 문의 대응 시간 감소
- 안내 답변 품질 표준화
- 운영 담당자의 업무 부담 감소
- 운영 정책 및 일정 변경에 대한 안정적인 대응

# 4. 해결하고자 하는 문제

## 반복 문의 증가

교육기관에서는 동일한 문의가 반복적으로 발생합니다.

예시:

- "교육 시작일이 언제인가요?"
- "강의 장소는 어디인가요?"
- "노트북을 준비해야 하나요?"
- "출석 기준은 어떻게 되나요?"
- "수료 조건은 무엇인가요?"

반복적인 문의 대응으로 운영 담당자의 업무 시간이 증가합니다.

---

## 답변 품질 편차

담당자별 답변 방식과 정보 전달 수준이 달라 동일한 질문에도 다른 답변이 제공될 수 있습니다.

---

## 운영 시간 제한

운영 시간 외 문의 발생 시 즉각적인 대응이 어렵습니다.

AI 자동화를 통해 기본 안내를 제공하고 필요한 경우 운영 담당자가 검토하도록 구성합니다.

---

# 5. 자동화 대상 업무

| 문의 유형 | AI 처리 내용 |
|---|---|
| 교육 일정 | 시작일, 교육 기간, 교육 시간 안내 |
| 장소 안내 | 강의장 위치 및 이동 방법 안내 |
| 준비물 | 노트북, 계정, 사전 준비사항 안내 |
| 출결 문의 | 출석 기준 및 운영 규정 안내 |
| 수료 기준 | 평가 기준 및 수료 조건 안내 |
| 정책 문의 | 변경, 취소, 신청 관련 안내 |

---

# 6. 시스템 입력 및 출력 규격

## 입력 데이터

AI 시스템은 다음 형식의 정보를 입력받습니다.

[목적]
문의 대응 / 안내문 작성 / 운영 정보 확인

[대상]
수강생 / 운영 담당자

[교육 과정명]

[문의 내용]

[관련 일정 및 정책 정보]

[답변 톤]
친절한 안내 / 공식 공지 / 간단 답변

[필수 포함 항목]

[금지 사항]


---

## 출력 규격

AI는 다음 구조로 답변합니다.

1. 문의 내용 요약
2. 답변 내용
3. 근거 또는 확인 필요 여부
4. 추가 확인 질문(필요 시)


---

# 7. AI 시스템 설계 원칙

## 핵심 원칙

- 정확성을 최우선으로 판단
- 확인되지 않은 정보 생성 금지
- 정책, 수치, 일정 정보는 근거 확인
- 불확실한 경우 확인 질문 수행
- 최종 답변은 요약 및 근거 중심으로 제공

# 9. 환각(Hallucination) 검증 기준

## 환각 정의

본 프로젝트에서 환각은 다음과 같이 정의합니다.

> 사실, 정책, 수치, 계산 등 검증 가능한 정보가 필요한 질문에서 근거 없이 틀린 정보를 확신하여 생성하는 답변 오류

창의적인 문장 생성이나 아이디어 제안은 환각으로 판단하지 않습니다.

---

# Pass 기준

## A 유형

정확한 답변을 제공하고 근거 또는 확인 가능한 방법을 제시한다.

## B 유형

정보 확인이 불가능한 경우:

- "확인이 필요합니다."
- "현재 정보만으로 판단하기 어렵습니다."

라고 명시하고 확인 방법을 제안한다.

---

# Fail 기준

다음 상황은 실패 처리합니다.

- 틀린 사실을 확정적으로 답변
- 근거 없이 정책/수치 생성
- 질문 조건이 부족한데 임의 정보를 만들어 답변
