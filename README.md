# 🧠 LLM 기반 이동 전략 설계 시스템 + 모델 비교 + 실행 로그 통합 보고서

---

# 0. Executive Summary

본 문서는 대전정보문화산업진흥원 이동 문제를 대상으로  
3종 이상의 LLM을 비교 평가하고,  
실제 10턴 이상 대화 로그 및 프롬프트 설계, 검증 전략을 통합하여

👉 **“LLM을 단순 챗봇이 아닌 의사결정 시스템으로 설계하는 방법”**을 검증한 결과물이다.

---

# 1. 업무 과업 정의 (Criterion 1)

## 🎯 Task Definition

- **업무 과업:** 대전정보문화산업진흥원까지의 대중교통 이동 전략 생성
- **타겟 사용자:**
  - AI 네이티브 교육 과정 수강생
  - 대중교통 기반 출퇴근 사용자
- **입력:**
  - 출발지
  - 시간대 (출근/평일/야간)
  - 제약조건 (환승 최소, 속도 우선 등)
- **출력:**
  - 경로 2~3개
  - 환승 전략
  - 시간대별 리스크
  - 비교 기반 의사결정 구조

# 3. 모델 비교 평가 (Criterion 3, 9)

## 📊 평가 기준

- 정확성
- 실용성
- 구조화
- 간결성
- 환각 위험 (역점수)

## 📌 비교 결과

| 모델 | 정확성 | 실용성 | 구조화 | 간결성 | 환각억제 | 총점 |
|------|--------|--------|--------|--------|----------|------|
| Gemini 3 Flash | 3.5 | 3.5 | 3.0 | 4.5 | 3.5 | 18.0 |
| Claude Haiku 4.5 | 3.0 | 2.5 | 3.5 | 4.0 | 4.5 | 17.5 |
| GPT-5.4 mini | 4.0 | 4.5 | 4.5 | 2.0 | 3.0 | 18.0 |

---

## 📌 해석

- Gemini → 빠르지만 전략 부족
- Claude → 안전하지만 실행력 부족
- GPT → 전략적 구조 가장 우수

# 5. 실행 로그 (10턴 이상) (Criterion 6, 15)

## 🧾 핵심 로그 요약

- Turn 1~3: 정상 경로 탐색
- Turn 4: 존재하지 않는 직행 버스 환각 발생
- Turn 7: 비교 구조 붕괴
- Turn 10: 막차 정보 단정 오류

---

## ❌ 주요 실패 유형

- 환각 (직행 버스 존재)
- 문맥 유지 실패
- 비교 구조 붕괴
- 시간 정보 단정 오류

---

## ✅ 개선 결과

- 경로 2개 유지
- 환각 감소
- 구조 안정화

---

# 6. 환각 검증 질문 (Criterion 5, 10, 11)

## ❓ Q1: 해당 노선 실제 존재하는가?
→ FAIL

## ❓ Q2: 시간 정보 출처가 있는가?
→ FAIL

## ❓ Q3: 대안 경로 2개 이상인가?
→ PASS

## ❓ Q4: 조건 유지되는가?
→ FAIL

## ❓ Q5: 비교 구조 존재하는가?
→ PASS

---

## 📌 판정 기준

- PASS: 외부 근거 또는 구조 유지 확인
- FAIL: 모델 내부 생성 단정 정보
