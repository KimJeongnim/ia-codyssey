# 🧠 LLM 이동 전략 시스템 설계 문서

---

## 1. 목표 (Criterion 7~12)

- 단순 경로 안내 → 의사결정 시스템 변환
- 환각 최소화
- 선택 가능한 구조 제공

---

## 2. System Prompt

당신은 대중교통 이동 전략 설계 전문가이다.

규칙:
- 불확실 정보 단정 금지
- 경로 2개 이상 제공
- 시간대 반영
- 환승 전략 포함
- 선택 구조 생성

---

## 3. User Prompt

출발지: {input}
시간대: {input}
조건: {input}

→ 경로 2~3개 + 비교 + 리스크 출력

---

## 4. Few-shot (Criterion 4)

### 모호 입력 대응
User: "빠르게 가는 방법"
→ 출발지 확인 요청

---

## 5. 프롬프트 개선 이력 (Criterion 8)

v1: 단순 경로
v2: 환승 추가
v3: 의사결정 구조화

---

## 6. 운영 전략 (Criterion 13~15)

### 무료 모델 대응
- 구조 강제 출력
- step-by-step 분해

### 정책 변화 대응
- 외부 데이터 분리
- RAG 구조

### 문맥 유지
- 상태 요약 유지
- constraint lock 적용

---

## 7. 환각 정의 (Criterion 10)

- 사실과 다른 “단정 정보 생성”
- 창의적 생성과 구분됨

---

## 8. 결론

LLM 성능 = 모델이 아니라 구조 문제
